Buttons

The following components pertain to the use of buttons within the system.

For All

This is the button component used for all screens (desktop, tablet, and mobile). It is displayed with a dark (#202022 in the color scheme) text when inactive, and light text (#f3f3f3) when hovered.


Usage & Best Practices

Buttons should be used sparingly. The portfolio site uses them for case studies and for additional content the user may need to navigate to a new page in order to see (see viewing images on mobile in the case study patterns). The current portfolio site does not use any forms, but these buttons would be appropriate to use in the event one wants to include forms on their webpage. The buttons should always be rounded and ideally justified right within their container.

Accessibility

Ensure that the buttons use the pink, or light, background with dark text for accessibility. Button text should only be light when a user hovers over the button.


Code Example

    
        <button type="button" class="btn btn-light">Light</button>
